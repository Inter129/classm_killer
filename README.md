# ClassM 우회하기 (하드보안관)

이 문서는 ClassM 모니터링 소프트웨어를 우회하는 다양한 방법을 제공합니다.

> [!IMPORTANT]
> 본 문서의 내용을 따라 수행하며 발생하는 모든 책임은 사용자 본인에게 있습니다. 계속 진행하기 전에 이를 충분히 숙지해 주시기 바랍니다.

## 시작하기 전에

### 필요 사항

- ClassM이 설치된 Windows 기반 컴퓨터
- 뇌와 문해력

### 주의사항

1. 학교, 회사, 기관의 정책에 위반되는 행위에 해당할 수 있습니다.
2. 본인의 책임 하에 사용하시기 바랍니다.

## Version 3.X 우회 방법 (2025.11.09 기준 최신 버전)

<details>
<summary>펼치기/접기</summary>

### 적용 버전

ClassM 3.0에서 테스트 및 검증된 방법입니다.

### Method 1: ProcessHacker를 이용한 프로세스 중지

#### 준비물

- ProcessHacker(또는 유사 프로그램) 다운로드 및 설치 (필자 기준 ProcessHacker는 학교pc에서 차단)

#### 실행 방법

1. ProcessHacker(또는 유사 프로그램)를 관리자 권한으로 실행합니다.
2. 프로세스 목록에서 ClassM 관련 프로세스를 찾습니다.
   예) classm 폴더 내의 프로세스들
3. 각 프로세스를 우클릭하여 **"Suspend"**를 선택합니다.
4. 모든 ClassM 프로세스가 일시중지 상태가 되면 모니터링이 중단됩니다.

#### 효과

- 사용자의 활동이 모니터링되지 않습니다.
- ClassM의 화면 제어 기능이 작동하지 않습니다.

### Method 2: 화면 잠금 우회

#### 현재 상태

현재까지 확인된 우회 방법이 없습니다.

#### 기여 요청

새로운 우회 방법을 발견하셨다면 [기여하기](#기여하기) 섹션을 참고하여 공유해 주세요.

</details>

## Version 2.X 우회 방법

<details>
<summary>펼치기/접기</summary>

### 적용 버전

ClassM 2.0에서 테스트 및 검증된 방법입니다.

### Method 1: ClassM Killer 프로그램 사용 (권장)

#### 다운로드

[GitHub Releases](https://github.com/Inter129/classm_killer/releases/tag/dev)에서 `classmkiller_b1.exe`를 다운로드합니다.

#### 실행 방법

1. 다운로드한 `classmkiller_b1.exe` 파일을 실행합니다
2. 프로그램이 백그라운드에서 자동으로 동작합니다

#### 동작 원리

- `classm_client.exe` 프로세스를 지속적으로 감지하고 종료합니다
- ClassM 클라이언트가 초기화되기 전에 종료하여 모니터링을 차단합니다
- 사용자의 활동이 방해받지 않습니다

#### 장점

- 자동화되어 사용이 간편합니다
- 지속적인 우회가 가능합니다

### Method 2: ProcessHacker를 이용한 프로세스 중지

#### 준비물

- [ProcessHacker](https://processhacker.sourceforge.io/) 다운로드 및 설치

#### 실행 방법

1. ProcessHacker를 관리자 권한으로 실행합니다
2. 프로세스 목록에서 ClassM 관련 프로세스를 찾습니다
3. 각 프로세스를 우클릭하여 **"Suspend"**를 선택합니다

#### 효과

사용자의 활동이 모니터링되지 않으며, ClassM의 제어 기능이 작동하지 않습니다.

### Method 3: 화면 잠금 임시 우회

#### 주의사항

이 방법은 임시적인 우회 방법으로, 지속적인 반복 작업이 필요합니다.

#### 실행 방법

1. `Win + R` 키를 눌러 실행 창을 엽니다
2. `taskkill /f /im classm_client.exe` 명령어를 입력하고 Enter를 누릅니다
3. 1~2초마다 `Win + R` → `Enter` 키를 반복합니다

#### 효과

화면 잠금 상태를 임시적으로 우회할 수 있습니다.

#### 한계

- 수동으로 반복해야 하므로 불편합니다
- 완전한 우회가 아닌 임시적인 방법입니다

</details>

## 문제 해결

### 자주 묻는 질문 (FAQ)

#### Q1. 프로그램이 실행되지 않아요

**원인 분석:**

- 관리자 권한 부족
- 백신 프로그램의 차단

**해결 방법:**

1. 파일을 우클릭하여 "관리자 권한으로 실행"을 선택합니다
2. 백신 프로그램의 예외 목록에 추가합니다

#### Q2. ProcessHacker에서 프로세스가 보이지 않아요

**원인 분석:**

- ClassM이 실행되지 않은 상태
- 프로세스 이름이 변경되었을 가능성

**해결 방법:**

1. ClassM이 실행 중인지 확인합니다
2. "classm"으로 검색하여 관련 프로세스를 모두 확인합니다

#### Q3. 우회 후 다시 모니터링이 시작돼요

**원인 분석:**

- ClassM 서비스가 프로세스를 재시작함

**해결 방법:**

- Method 1 (Killer 프로그램)의 창을 종료하지 않습니다.
- 또는 모든 관련 프로세스를 직접 Suspend 상태로 유지합니다
